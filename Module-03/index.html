Concatenation

We have (up to this point) used the standard string syntax utilizing the single quotes and the double quotes. You may ask what the difference is, and (to be honest) there really isn't any. Most developers don't really care which one you use as long as you're consistent. It is more common to use single quotes than double quotes.

Often (like, a LOT) we want to create a dynamic string utilizing a value we've concocted during our application's progress. Perhaps it's data from a source like a user or an API. Or perhaps it's a calculation we now want to output. Either way, we need some method to output our value in combination with a string. That is why we have concatenation. Concatenation is of merging 2 or more values together. This is definitely where a demonstration helps to illustrate the point:

Example: Concatenation 
                    
                      // Concatenation
                      let price = 17;
                      let subtotal = 'The price of the product is $' + price;
                      let withTax = 'The tax is $' + (price * 0.10);
                      let total = 'The total is $' + (price * 1.10) + '. Thank you for your patronage.';

String Interpolation

Concatenation is cool, but string interpolation using the string template is by far MORE cool! It allows us to evaluate an expression inline with a string. Many languages support this feature using their double quote characters, however, JavaScript utilizes a third quote character instead; the backtick!

The ` backtick character can be found sharing space with the ~ tilde character. A set of these denotes you want to use string interpolation. In order to evaluate an expression, you simply wrap it in a set of {} curly braces and prefix it with a $ dollar sign:

Example: Concatenation 
                    
                      // Concatenation
                      let price = 17;
                      let subtotal = 'The price of the product is $' + price;
                      let withTax = `The tax is ${price * 0.10}`;
                      let total = `The total is ${price * 1.10}. Thank you for your patronage.`;

Library Functions

What the Func is a Function?

Functions are essential to DRY code. They allow us encapsulate a block of logic to be reused over and over again as we need. They also allow us to abstract logic so it can meet various use cases. This is achieved through the use of parameters. Parameters are essentially variables scoped to the function. The developer using the function then passes arguments whose values will populate the variables allowing for value specific logic to be executed.

Objects have functions available to them as well. These are often known as methods. Functions defined in JavaScript are ALWAYS apart of an object. When you create a function within the root of a file, your function is added to the Window object, an object created by the DOM API in your browser. The same applies to your variables. They become properties on the Window object.

Example: The Window Object 
                  
                    // Below is are a variable and a function
                    var x = 5;
                    function add (a, b) {
                      return a + b;
                    }
Output

As you can see, .x is a property of Window and .add() is a method on Window. It is important to understand that this only applies to variables scoped globally or with var, and to functions that are globally scoped.

Without functions or methods we would have a ghastly amount of repetitive code in our applications. Before functions, older programming languages used subroutines and goto statements. The compiler would just reference that subroutine based on its line number or namespace. This was problematic and it is definitely better that we now use functions.

Casting

Because JavaScript is a loose-typed language, it is important to understand that comparisons can lead to unexpected results. We've seen this when using the loose equals == or the loose !=. These will evaluate the values of two items, but not their data type. This leads to "5" == 5, and true == 1. You may feel that as long as we use the strict equals to === we should never have an issue, but unfortunately that's not true. We can't always guarantee the data we're receiving from users, APIs, or third-party libraries. To fix this and create a normalized environment we can use casting.

The casting methods can be found on the Window object, but are available to us without the need to type window.method(). The 2 most common are Number() and String(). These convert their argument to either a number or string if it can be converted. String will always convert, however Number may return NaN.

Example: String and Number Casting 
                  
                    // String will convert anything to a string
                    let bool = String(true);
                    typeof bool; // string

                    let str = String(5.45);
                    typeof str; // string
                    console.log(str); // "5.45"

                    // Number will attempt to convert to a number
                    let num = Number("0.46");
                    typeof num; // number

                    num = Number("shaun");
                    typeof num; // number
                    console.log(num); // NaN (Not a Number)

Browser Dialogue Functions

These next 3 functions aren't used often but are important to know about. They are the dialogue functions alert, prompt, and confirm. Their purpose is to provide a predefined browser based method for interacting with the user.

Command Description Common Use Case
alert() Provides a message to the user. Used to notify the user in a guaranteed method.
prompt() Provides a message and a way to collection information from the user. Used to gather imperative information from the user.
confirm() Provides a boolean choice to the user. Used most often to confirm if the user wants to do something destructive.
Example: Dialogue Functions 
                  
                    // alert
                    alert('The falcon flies south for the winter!');

                    // prompt()
                    let response = prompt('How old are you?');

                    // confirm()
                    let choice = confirm('Are you sure?');

Math

We may want to run away from math, but the unfortunate reality is we will likely have to use it. JavaScript, like many languages, comes with a built-in math library. There are the standard operations, as illustrated in the table below:

Operator Description Example
+ Addition is performed using the plus sign. This operator will add the left side to the right. Note that it is also the concatenation operator which will combine the left and right sides. 5 + 5; // 10
- Subtraction is performed with the minus sign. This operator will subtract the right side from the left side. 10 - 5; // 5
/ Division is performed using the forward slash. This operator will divide the right side into the left side. 10 / 5; // 2
* Multiplication is performed using the asterisk. This operator will multiply the left by the right. 10 * 5; // 50
% Modulo is used to return the remainder from a division operation. The percent sign is used for this operation. 8 % 3; // 2
Calculations in JavaScript are performed using floating point notation. This can lead to some bizarre results. For example, 3.3 = 1.0999999999 as opposed to 1.1.

In order to perform complex math such as ceil, floor, square, and round, you must use the JavaScript Math library API. It is already available to you. To use it, simply access the library with the symbol name Math, then use the . dot notation to access a method:

Example: Math Functions 
                  
                    // Square Root
                    let sqrt = Math.sqrt(16); // 4

                    // Round Up
                    let ceil = Math.ceil(10.5); // 11

                    // Round Down
                    let floor = Math.floor(10.5); // 10

                    // Round based on midpoint
                    let round = Math.round(10.5); // 11
                    round = Math.round(10.4); // 10

                    // Random number
                    let random = Math.random();
                    // Returns a number between 0 and 1

String

There are numerous string methods, but we tend to only use a few. We can make a string uppercase or lowercase. We can count a string. We can replace text within a string. We can also search a string for a specific pattern. In addition we can trim whitespace form the beginning or end of a string.

"".toUpperCase() and "".toLowerCase() change the string to their respective cases. "".trim() will remove all whitespace at the beginning and end of a string. "".replace() takes 2 arguments; the pattern and the replacement string. "".indexOf() returns the starting index number of where the passed pattern was found. "".match() takes a pattern for an argument and will return an array with all the matches, the index where it was found, and any groups you may have defined. .replace(), .indexOf(), and .match() can all take regular expression patterns as their first argument.

Example: String Functions 
                  
                    // Uppercase
                    let up = "Shaun McKinnon".toUpperCase();
                    // "SHAUN MCKINNON"

                    // Lowercase
                    let down = "Shaun McKinnon".toLowerCase();
                    // "shaun mckinnon"

                    // Trim
                    let trim = "  Shaun McKinnon  ".trim();
                    // "Shaun McKinnon"

                    // Replace
                    let rep = "Shaun McKinnon".replace("Shaun", "Dave");
                    // "Dave McKinnon"

                    // Finding first occurrence of
                    let first = "Shaun McKinnon".indexOf("McKinnon");
                    // 6

                    // Finding matches
                    let matches = "Shaun McKinnon".match("McKinnon");
                    // ["McKinnon", index: 6, input: "Shaun McKinnon", groups: undefined]

Custom Functions

Writing a Custom Function

Creating our own functions gives us the power to modularize code that we intend to repeat or scope code to a namespace. Just like variables in JavaScript, functions don't need to be data typed. In addition, we don't have to define what they're returning.

The syntax for a function in JavaScript is simple: the keyword function, the symbol name, a list of parameters if you want some, and a block of logic to execute. In fact, even the symbol name is optional, though this would make our function an anonymous function. Anonymous functions are quite common, especially when used as a value being stored in a variable or passed to another function as an argument.

Example: Custom Function Syntax 
                  
                    // Standard Function
                    function myFunc (param1, param2) {
                      // my logic block
                      console.log("Hello World");
                      return "bob";
                    }

                    // Anonymous Function
                    let myFunc = function(param1, param2) {
                      // my logic block
                      console.log("Hello World");
                      return "bob";
                    }
Practice 07: Custom Functions

Calling a Custom Function

Calling custom functions is the same as calling any function. An important concept to learn in JavaScript is that functions are just function definitions until you use the execution operator (). The double parenthesis execute the function definition. Otherwise we are able to pass the function definition around as a value.

Example: Calling Functions 
                  
                    // Standard Function
                    function myFunc (param1, param2) {
                      // my logic block
                      console.log("Hello World");
                      return "bob";
                    }

                    myFunc(arg1, arg2);

                    // Anonymous Function
                    let myFunc = function(param1, param2) {
                      // my logic block
                      console.log("Hello World");
                      return "bob";
                    }

                    myFunc(arg1, arg2);

Parameters & Arguments

Parameters

Parameters are simply placeholders for expected values. They allow you to imply a value in various spots in your logic without needing the value before hand. Parameters are not data typed which means the arguments will need to be sanitized

Example: Parameters 
                    
                      // Parameters
                      function myFunc (str, num, obj, arr, func) {
                        console.log(str, num, obj, arr, func);
                      }
                      // Parameters are just symbol names (labels);
                      // they are placeholders for data values. They
                      // are not data typed!

Arguments

Arguments are the values that will replace the parameters within the function. Because parameters are not data typed, it is important to perform sanitization and validation on the arguments.

Example: Arguments 
                    
                      // Arguments
                      myFunc("Bob", 40, {a: 2, b: 3}, [1, 2, 3], myfunc);
                      // Arguments can be any data type including
                      // a function definition!

Callbacks

Last week we discussed how arrays have many methods available to them, however most took a callback as an argument. A callback, simply put, is a function definition that has been passed as an argument. Once you have the definition, you can choose to call it inside your original function as you see fit. Because JavaScript is asynchronous, callbacks are crucial to queuing operations. For example, when an API has returned a value, we can call the callback to add the values to the DOM.

Due to the complexity of callbacks, we'll use a simple built-in method and provide it a callback so we can get a general idea of how it works.

Below, we're using the array method map. Map is an interative method, meaning it will loop over all the elements in the array. Essentially, .map iterates over each element and passes it to a provided callback function. That function can mutate the value, output it, whatever it wants. If it returns a value, that value will take the spot of the passed element:

Example: Callbacks 
                  
                    let myArr = [1, 2, 3, 4, 5];

                    // Map with an anonymous function
                    let newArr = myArr.map(function (ele) {
                      return ele + 1;
                    });
                    // newArr = [2, 3, 4, 5, 6]

                    // Map with a named function
                    function output (ele) {
                      return ele + 1;
                    }

                    let newArr2 = myArr.map(output);
                     // newArr2 = [2, 3, 4, 5, 6]
