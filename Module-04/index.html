Object Recap

Objects, like in most languages, are a way for us to encapsulate code under one idea. Generally an object is best described using a noun (a  person, place, or thing).

Properties are basically descriptive words (adjectives) that can be used to define properties or attributes about an object. Right, we see an JavaScript object literal defined with 3 properties: name, age, and interests. 

                  
                    let person = {
                      name: "Shaun",
                      age: 39,
                      interests: ["programming", "Diablo", "cage fighting"]
                    }

Methods are actions an object can perform. For example an object of type Shaun will likely be able to talk and walk. 

let person = {
  name: "Shaun",
  age: 39,
  interests: ["programming", "Diablo", "cage fighting"]
  talk: function () {
    console.log("I'm 39");
  },
  walk: function () {
    posX += 5;
  }
}

Primitives & Objects

The statement "Everything's an OBJECT in JavaScript" is a misnomer. MOST things in JavaScript are objects. Or they're wrapped by an object if you attempt to use the access operator "." when accessing a property or method. That last bit is what confuses people into thinking that everything's an object. If a string isn't an object, then why can I call "hello".length? The reason you can access the property is because as soon as you use the "." the interpreter takes your string, instantiates a new String() object passing it as the argument to the constructor; new String("hello"); which then returns a string object allowing you to call "hello".length. That's a lot of magic but helpful to you. The current primitives are booleans, strings, numbers, null, and undefined.

Remember, primitives (booleans, strings, numbers, null, and undefined) are not objects but they can can be implicitly cast into an object if you attempt to access a property or method on them.

Built-In Objects

Data Objects

Objects are an integral part to JavaScript. In JavaScript, ALMOST everything is an object. That's very cool if you consider that objects give you properties and methods. It's like having extra functionality on every variable, function, string, number, and array (for example) that you place in your code!

  • String()
  • Number()
  • Boolean()
  • Array()
  • Object()
  • Function()

These are the data objects. Each can be instantiated to create new objects, but they're not to be confused with classes in other languages. JavaScript objects exist immediately. They are not just a blueprint. This approach allows for dynamic property and method modification/creation during execution. For example, you can add new properties to the parent String() object. Those properties will then cascade to objects that have been created using the String object.

ALL objects in JavaScript are inherited from the base Object(). This means that they have access to all the properties and methods that the base object has available.

We can see what properties an object has available by using the Object() method .getOwnPropertyNames() and passing an object as an argument.

DRYing Up Objects: Object Constructors

So far we have built very specific objects. person was an object that contained several defined properties, but was specific to only one person. 

let person = {
  name: "Shaun McKinnon",
  age: 40,
  dob: "1978-12-22",
  bio: function () {
    return `${this.name} is ${this.age} years old. They were born on ${this.dob}.`;
  }
};

But what if we wanted to add another person?

Object Constructors are simply just a standard function. We can use these to create a blueprint (just like classes do) for what we want an object to look like -- any necessary properties it may need to contain.
!!! REMEMBER: Properties in JavaScript can contain any value type INCLUDING function definitions. !!!

In order for us to utilize object constructors we need to understand the concept of this. this is a powerful keyword in object oriented programming. In objects this represents the owner object, which in the case of an object constructor, would be the wrapping function.

REMEMBER: MOST things in JavaScript are already objects. This includes functions. 

// This is an object constructor. It is just a simple
// function. Remember: Most things in JavaScript are
// an Object. This includes functions!
function MyObjectConstructor (prop1, prop2, prop3) {
  // 'this' refers to MyObjectConstructor
  this.prop1 = prop1;
  this.prop2 = prop2;
  this.prop3 = prop3;
}
