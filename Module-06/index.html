Introducing the DOM

What is the DOM?

Introducing....

So far we have learned a lot about objects: properties; methods; inheritance; passing by reference. This foundational knowledge will help you better understand the magic that is the DOM.

The DOM is simply an object. It is no different than the objects you create in OOP. The DOM represents our entire HTML page. It provides, like objects do, properties and methods that allow us to create, read, update, and delete various pieces of our HTML page.

The DOM is not exclusive to JavaScript, but often the two are used in tandem. The browser's JavaScript API provides a way for you to work with the DOM. On page load, the JavaScript API creates a new version of your API page. It first creates the document object then proceeds to populate it with children nodes that represent the other HTML elements you have on the page. Each of these is a different object type, but all inherit from the main object types, Element, Node, and EventTarget.

Understanding the DOM structure allows you to manipulate it with a programming language like JavaScript more effectively. The DOM, however, is not explicitly a JavaScript thing. It was made to be independent of any programming languages.

DOM Selection Methods

Using CSS Selectors

The DOM API provides several methods for selecting HTML elements on the page. HTML Elements are the tags that make up your HTML document. The DOM API creates HTMLElement objects to represent and interact with those tags. The selectors allow us to search for the HTMLElement that corresponds to a tag. The best part is we use CSS selectors as search parameters to retrieve the HTMLElement objects.

The two standard methods for selecting HTMLElements are document.querySelector() and document.querySelectorAll(). These methods, while similar, return different results. document.querySelector() will return a single HTMLElement object where as document.querySelectorAll() will return a NodeList containing one or more HTMLElement objects.

Both DOM selector methods take a single argument: a CSS selector string. These are the same selectors your use to write CSS styles. These selector strings help the methods search and find what HTMLElement you're attempting to select.

The more precise your selector string the faster the element will be returned to query.

Inspecting an Element

All the major browsers provide a way to inspect an element. You simply right click on the element you want to inspect and choose inspect. The browser (in the developer tools window) will show you the element HTML allowing you to quickly see what tag it is, what attributes it contains, and any children that may be nested within the element.

DOM Walking With Scope

The document object translates our entire HTML page into a collection of HTMLElements. The document object itself inherits from the Node object which provides us the methods .querySelector() and .querySelectorAll(). All HTMLElements also inherit from the Node object. Which your Sherlock deducing skills should tell you ALL HTMLElements have the methods .querySelector() and .querySelectorAll()! So what exactly does this mean? .querySelector() and .querySelectorAll() are methods wrapping search algorithms. The object their called on is their scope. That means if we select an HTMLElement then call .querySelector() or .querySelectorAll() it will look for HTMLElements, matching the passed CSS selector string, nested within that HTMLElement.

Working With an HTMLElement

Accessing/Changing Content

All HTMLElements have a property called .textContent. In addition, all HTMLElements that are form elements have a a property called .value. These properties are responsible for holding the very text we see on our page. If we want to change that text we merely access the appropriate property and modify its current content.

Changing Styles

HTMLElements have several properties. For example, you can change any of their attributes by simply accessing the attribute as a property. These include name, id, title, href, src, alt, etc... The best part is we're not limited there. We can also access the style property. The style property, however, has some bonus capabilities. It itself is an object containing properties for every style attribute available to the HTMLElement. By accessing these attributes we can dynamically modify them!

Working With NodeLists

Nodelists

NodeList objects are collections of Node instances. An HTMLElement is a node instance. When we want to select all items matching a CSS selector we use .querySelectorAll(). This will return a NodeList containing all the nodes that match the passed CSS selector. Using standard iterative structures like for loops or the .forEach method we can iterate through our list and perform operations on the returned node.

Practice 10: NodeList(s)

NodeList as Array Caveats

NodeLists are similar to Arrays, however, there are some caveats to treating them that way. For example, not all Array methods are available on NodeLists. .filter(), .map(), and .reduce(), for example, will cause an error if you attempt to use the as they are not methods of the NodeList object.
